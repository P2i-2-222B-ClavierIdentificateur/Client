############## Structure du Projet ######################

-les fichiers .java sont dans le dossier src.

-les fichiers .class sont dans le dossier bin.

-le fichier resultats-compiles.csv sert à centraliser les résultats en temps 
pour le mot de passe commun (pour le moment "password") en attendant que l'on
implémente la base de donnée.

-lors de vos tests les résultats sont écrits dans un fichier resultats.csv.


############### Consignes générales  ###################

-Ne modifiez pas directement la branche master, on approuvera les changements 
ensemble pendant les séances de projet.

-Si vous voulez apporter une modification au code ou à un autre fichier ou même 
ajouter un nouveau fichier, utilisez votre propre branche du projet (voir la 
partie concernant l'utilisation de github.

-Vous pouvez ajouter des objectifs au projet en allant dans l'onglet issues puis 
milestones de github.



############## Utilisation de Github ###################

-Pour récupérer le projet utilisez le lien que vous fournit le site losque 
vous cliquez sur "Clone or Download".

-Vous pouvez télécharger le client de github pour Windows et Macs. 

-Pour créer votre propre branche pour travailler sur le projet, cliquer sur 
le menu déroulant "Branch : <nom_d'une_branche>" et entrez le nom de la branche
que vous souhaitez créer.

-Lorsque vous souhaitez soumettre vos modifications au reste du groupe, allez sur
la page du projet et cliquez sur "New pull request" et utilisez les menus déroulants 
pour obtenir la configuration suivante :

	Base : Master  ...  Compare : <votre_branche>

Ajoutez ensuite un titre et une description à votre modification et cliquez sur 
"Create pull request".

!!!!!!!!!! Attention : étant donné que tout le monde à tous les droits sur le projet
 ne faites attention à ne pas cliquer sur "merge pull request" sur la page suivant,
 cela modifierait la branche master !!!!!!!!!!!!!!!!!
